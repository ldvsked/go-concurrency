# Go Concurrency

Коллекция проектов на Go, посвященных конкурентному программированию. Репозиторий демонстрирует использование goroutines, channels, примитивов синхронизации из пакета `sync`, обработку системных сигналов и реализацию потокобезопасного LRU-кэша с использованием Generics.

## Проекты

### 1. Timer (`/timer`)

Запускает заданное количество горутин, каждая из которых выполняется случайное время, после чего выводит результаты, отсортированные по убыванию времени выполнения.

**Особенности реализации:**

* запуск множества goroutines;
* синхронизация завершения через `sync.WaitGroup`;
* потокобезопасное сохранение результатов с использованием `sync.Mutex`;
* сортировка результатов перед выводом.

**Пример запуска**

```bash
go run ./timer 5 1000
```

где:

* `5` — количество горутин;
* `1000` — максимальное время выполнения в миллисекундах.

---

### 2. Square Generator (`/square_generator`)

Конвейер обработки данных, состоящий из двух конкурентно работающих этапов: генерации последовательности чисел и их возведения в квадрат.

**Особенности реализации:**

* взаимодействие горутин через каналы;
* использование однонаправленных каналов;
* конкурентное выполнение генератора и обработчика;
* корректное закрытие каналов.

**Пример запуска**

```bash
go run ./square_generator 1 5
```

где:

* `1` — начало диапазона;
* `5` — конец диапазона.

**Результат**

```text
1
4
9
16
25
```

---

### 3. Ticker (`/ticker`)

Асинхронный таймер, выводящий сообщение через заданный интервал времени до получения сигнала завершения программы.

**Особенности реализации:**

* собственная реализация периодического выполнения без использования `time.Ticker`;
* обработка сигналов `SIGINT` и `SIGTERM`;
* корректное завершение работы программы.

**Пример запуска**

```bash
go run ./ticker 2
```

где:

* `2` — интервал между сообщениями в секундах.

---

### 4. LRU Cache (`/lru_cache`)

Потокобезопасная реализация LRU-кэша с использованием Generics.

**Особенности реализации:**

* алгоритм **Least Recently Used (LRU)**;
* операции `Get` и `Set` со сложностью **O(1)**;
* использование Generics;
* потокобезопасность при конкурентном доступе;
* поддержка методов `Set`, `Get` и `Clear`;
* модульные тесты для проверки логики вытеснения элементов.

## Запуск

Запуск проектов:

```bash
go run ./timer <goroutines> <max_sleep_ms>
go run ./square_generator <start> <end>
go run ./ticker <interval_seconds>
```

Запуск тестов:

```bash
go test ./... -v
```
